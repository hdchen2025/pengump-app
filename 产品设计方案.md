## 一、核心玩法设计

### 1.1 游戏概述

打企鹅是一款以"愤怒的小鸟"为原型的休闲弹道射击游戏。玩家操控弹弓，将可爱的企鹅发射出去，撞击浮冰上的目标（冰块），击碎所有冰块即可通关。游戏融合了经典8-bit像素艺术风格与简单直观的触控操作，适合全年龄段用户。

### 1.2 详细游戏机制

#### 发射流程

1. **选择企鹅**：从底部企鹅队列中选择一只待发射的企鹅
2. **拉弓瞄准**：手指按住企鹅，向后拖拽（类愤怒的小鸟操作）
3. **观察轨迹**：松开手指前，系统显示虚线抛物线轨迹预测
4. **发射**：松手，企鹅沿弹道飞出
5. **飞行与碰撞**：企鹅在空中飞行，撞击冰块造成伤害
6. **回合结算**：所有企鹅用完后，统计得分与通关条件

#### 弹道物理参数

| 参数 | 数值 |
|------|------|
| 最大拉力距离 | 120 像素（屏幕空间） |
| 最小拉力距离 | 20 像素 |
| 发射速度倍率 | 拉力距离 × 0.15 |
| 最大初速度 | 18 单位/帧 |
| 重力加速度 | 0.25 单位/帧² |
| 空气阻力系数 | 0.99（每帧速度衰减） |
| 弹弓皮筋弹性系数 | 0.4 |

#### 企鹅飞行距离与角度的关系

基础条件：拉力距离 = 100 像素，发射角度 = 45°

| 发射角度 | 理论飞行距离（相对值） | 实际飞行距离（像素） | 适用场景 |
|---------|---------------------|---------------------|---------|
| 15° | 0.52x | ~312 px | 擦边击中、精准控制 |
| 30° | 0.87x | ~522 px | 近距离精确打击 |
| 45° | 1.00x（基准） | ~600 px | 标准中距离 |
| 60° | 0.87x | ~522 px | 高弧线绕过障碍 |
| 75° | 0.52x | ~312 px | 超高弧线、后方冰块 |
| 90° | 0.18x | ~108 px | 几乎垂直下落 |

> **注**：实际距离受重力与空气阻力影响，以上为参考值。游戏内提供轨迹预测线辅助瞄准。

### 1.3 物理碰撞规则

#### 碰撞类型

| 碰撞对象 | 碰撞效果 |
|---------|---------|
| 企鹅 → 冰块 | 冰块减少耐久度；企鹅根据撞击角度反弹 |
| 企鹅 → 地面 | 企鹅停止运动，本回合结束 |
| 企鹅 → 边界（左/右） | 水平速度反转（反弹） |
| 企鹅 → 顶部边界 | 垂直速度反转 |
| 冰块 → 冰块 | 冰块之间产生连锁推动 |
| 企鹅 → 星星道具 | 星星被收集，得分加成 |
| 企鹅 → 弹弓框 | 企鹅弹回，不消耗回合 |

#### 碰撞伤害计算

```
伤害值 = 企鹅速度 × 质量系数 × 碰撞角度系数

- 企鹅速度：sqrt(vx² + vy²)，范围 0~18
- 质量系数：基础企鹅 = 1.0，金属企鹅 = 1.8，布企鹅 = 0.6
- 碰撞角度系数：正面撞击(0~30°) = 1.2，斜撞(30~60°) = 1.0，擦边(60~90°) = 0.6
```

#### 物理引擎配置

- 引擎：SpriteKit 内置物理引擎（2D刚体）
- 碰撞边界：屏幕左/右/上边缘（可穿透底部）
- 物理categories：企鹅、冰块、地面、边界、道具

### 1.4 关卡设计（15关）

#### 关卡难度分级

| 关卡范围 | 难度标签 | 冰块数量 | 特殊元素 | 目标分 |
|---------|---------|---------|---------|-------|
| 1-3 | 入门 | 3-5 | 无 | 500/800/1200 |
| 4-6 | 简单 | 5-8 | 裂纹冰块 | 1800/2500/3500 |
| 7-9 | 中等 | 8-12 | 裂纹冰块、爆炸冰 | 4500/5500/6500 |
| 10-12 | 困难 | 12-16 | 所有类型+移动平台 | 8000/10000/12000 |
| 13-15 | 精英 | 16-20 | 全元素+多层次结构 | 15000/20000/25000 |

#### 详细关卡设计

**第1关 - 企鹅的第一次飞行**
- 冰块：3个普通冰块（耐久度1），呈三角形排列
- 企鹅数量：3只
- 目标分数：500
- 教学提示："拖动企鹅瞄准，松手发射！"

**第2关 - 排列的艺术**
- 冰块：4个普通冰块，横向排列
- 企鹅数量：3只
- 目标分数：800
- 教学提示："利用弹跳可以一次击中多个！"

**第3关 - 高塔挑战**
- 冰块：5个（4个普通+1个裂纹），堆叠成塔
- 企鹅数量：3只
- 目标分数：1200

**第4关 - 星星的诱惑**
- 冰块：5个普通冰块
- 特殊：关卡中有3颗可收集星星（每颗50分）
- 企鹅数量：4只
- 目标分数：1800

**第5关 - 裂纹冰块**
- 冰块：6个（3个普通+2个裂纹+1个普通）
- 裂纹冰块特点：受到一次撞击后变为普通冰块，再受一次撞击才会碎裂
- 企鹅数量：4只
- 目标分数：2500

**第6关 - 冰山的秘密**
- 冰块：8个，模拟冰山形状（底部宽、上部窄）
- 企鹅数量：4只
- 目标分数：3500

**第7关 - 爆炸危机**
- 冰块：8个（4个普通+2个裂纹+2个爆炸冰）
- 爆炸冰：受到撞击后对周围1格范围的冰块造成额外50%伤害
- 企鹅数量：5只
- 目标分数：4500

**第8关 - 移动的靶子**
- 冰块：8个（部分在移动平台上）
- 移动平台：左右缓慢移动（速度0.5px/帧）
- 企鹅数量：5只
- 目标分数：5500

**第9关 - 三星通关**
- 冰块：10个
- 三星条件：⭐ = 6500分，⭐⭐ = 8000分，⭐⭐⭐ = 10000分
- 企鹅数量：5只
- 目标分数：6500（一星）

**第10关 - 坚固的堡垒**
- 冰块：12个，复杂多层结构
- 企鹅数量：5只
- 目标分数：8000

**第11关 - 天空中的冰**
- 冰块：14个，分布在不同高度（使用悬浮平台）
- 企鹅数量：6只
- 目标分数：10000

**第12关 - 倒金字塔**
- 冰块：16个，倒三角结构（上大下小）
- 企鹅数量：6只
- 目标分数：12000

**第13关 - 极限挑战**
- 冰块：18个，高密度多层结构
- 新增元素：反弹板（可将企鹅弹向任意方向）
- 企鹅数量：7只
- 目标分数：15000

**第14关 - 冰与火之歌**
- 冰块：18个
- 新增元素：火焰区域（企鹅经过会受到伤害，额外消耗一只企鹅）
- 企鹅数量：7只
- 目标分数：20000

**第15关 - 最终Boss：企鹅王的城堡**
- 冰块：20个+1个Boss核心（超高耐久）
- Boss核心：耐久度5，需要多次精准撞击
- 特殊机制：城堡有3层护甲，每层护甲被击碎后形态变化
- 企鹅数量：8只
- 目标分数：25000
- 通关奖励：限定皮肤"黄金企鹅"

### 1.5 得分系统与Combo机制

#### 基础得分

| 击碎对象 | 基础得分 |
|---------|---------|
| 普通冰块 | 100 分 |
| 裂纹冰块 | 150 分 |
| 爆炸冰块（引爆） | 200 分 |
| 收集星星 | 50 分/颗 |
| 收集金币 | 10 分/枚 |

#### Combo连击机制

Combo触发条件：单只企鹅在飞行过程中连续击中多个目标

| Combo等级 | 击中数量 | 得分倍率 | 显示特效 |
|----------|---------|---------|---------|
| 无Combo | 1 | 1.0x | 普通 |
| Combo ×2 | 2 | 1.5x | 绿色数字飘字 |
| Combo ×3 | 3 | 2.0x | 蓝色数字飘字+星星粒子 |
| Combo ×4 | 4 | 3.0x | 紫色数字飘字+火焰粒子 |
| Combo ×5+ | 5+ | 4.0x | 金色数字飘字+彩虹爆炸 |

#### 关卡星级评分

| 星级 | 条件 |
|-----|------|
| ⭐ | 达到目标分数 |
| ⭐⭐ | 达到目标分数 × 1.3 + 剩余企鹅 × 200 |
| ⭐⭐⭐ | 达到目标分数 × 1.6 + 剩余企鹅 × 500 + 未使用的bonus企鹅 |

### 1.6 体力/次数限制机制

#### 体力系统

| 参数 | 数值 |
|------|------|
| 最大体力 | 30 点 |
| 每5分钟恢复 | 1 点 |
| 每次挑战消耗 | 5 点 |
| 体力耗尽后 | 可观看广告恢复5点，或等待 |

#### 企鹅配额

- 每关开始时分配固定数量的企鹅（3-8只）
- 用完企鹅但未通关：挑战失败
- 剩余企鹅可折算为额外分数（每只200分）
- Bonus企鹅：特定关卡额外赠送的特殊企鹅，不计入正常配额

#### 付费体力方案

| 商品 | 内容 | 价格 |
|------|------|------|
| 小体力包 | +30体力（立即恢复） | ¥6 |
| 中体力包 | +100体力（立即恢复） | ¥18 |
| 大体力包 | +300体力（立即恢复） | ¥50 |
| 月卡 | 每日发放50体力 + 关卡体力消耗-20% | ¥30/月 |

## 二、像素艺术规范

### 2.1 像素画通用规范

#### 画布规格

| 类型 | 像素尺寸 | 显示尺寸（@2x） | 显示尺寸（@3x） | 使用场景 |
|------|---------|----------------|----------------|---------|
| 大精灵 | 32×32 | 64×64 px | 96×96 px | 企鹅、冰块、道具 |
| 中精灵 | 24×24 | 48×48 px | 72×72 px | 小型装饰、按钮图标 |
| 小精灵 | 16×16 | 32×32 px | 48×48 px | UI元素、状态图标 |
| 特大精灵 | 64×64 | 128×128 px | 192×192 px | Boss、Logo |

#### 颜色调色板规则

- **主精灵**：最多使用 **6色**（含透明色）
- **复杂精灵**：最多使用 **8色**（含透明色）
- **UI图标**：最多使用 **4色**（含透明色）
- 颜色格式：HEX 或 RGBA(255,255,255,255)

#### 通用调色板（企鹅系列）

| 编号 | 色值 | 名称 | 用途 |
|------|------|------|------|
| 透明 | - | transparent | 透明区域 |
| 黑 | #1A1A2E | Black | 轮廓线、眼睛 |
| 白 | #FFFFFF | White | 眼白、高光 |
| 主体色 | #4ECDC4 | Teal | 企鹅身体主色 |
| 深色 | #2C7873 | DarkTeal | 阴影、立体感 |
| 粉色 | #FF6B9D | Pink | 腮红、可爱元素 |
| 橙色 | #FFE66D | Orange | 嘴巴、爪子、星星 |
| 灰色 | #95A5A6 | Gray | 弹弓绳 |

#### 导出格式

- 格式：PNG（带透明通道）
- 色彩空间：RGBA
- 位深度：8-bit（每个通道）
- 文件命名：`{sprite_name}_{state}_{frame}.png`

## 五、技术方案

### 5.1 技术栈选择

#### 推荐方案：SpriteKit（Apple原生）

**优势对比**

| 维度 | SpriteKit（推荐） | Flutter |
|------|------------------|---------|
| 平台 | Apple全平台优化 | 跨平台（iOS/Android） |
| 物理引擎 | 内置2D物理引擎，开箱即用 | 需要集成Box2D/Chipmunk |
| 性能 | 原生Metal渲染，性能最佳 | 依赖Skia，性能略逊 |
| 开发效率 | 高（Xcode一键打包） | 中（需要配置环境） |
| 像素艺术支持 | 优秀（点阵渲染模式） | 良好（需要设置pixelRatio） |
| 学习成本 | 低（Swift学习曲线平缓） | 中（Dart语言） |
| App Store优化 | 原生支持 | 需要额外配置 |
| 长期维护 | Apple官方持续更新 | 社区驱动 |

**结论**：iOS独占游戏，选择SpriteKit能获得最佳性能与开发体验，且SpriteKit的2D物理引擎完美匹配弹道类游戏需求。

#### 备选方案：Flutter（仅在考虑Android移植时考虑）

- 如果未来有Android移植计划，可考虑Flutter
- 但对于iOS独占产品，SpriteKit更合适

### 5.4 像素素材获取策略

> **预算目标**：像素素材总花费控制在 **¥200以内**
> **核心原则**：能用代码生成的就不用素材，能用免费的就绝对不花钱

#### 5.4.1 素材分类策略

| 类别 | 策略 | 占比 | 说明 |
|------|------|------|------|
| UI元素、粒子特效 | **代码程序化生成** | ~40% | 无需外部素材，完全自研 |
| 背景场景（静态） | **免费资源** | ~35% | 利用免费图库 |
| 游戏精灵（企鹅/冰块） | **低价购买** | ~20% | 控制在¥200内 |
| 音效/BGM | **免费素材** | ~5% | 开源音频 |

#### 5.4.2 代码程序化生成素材（优先级最高）

以下素材完全通过代码/Swift绘制，**零成本**：

| 素材类型 | 实现方式 | 技术方案 |
|---------|---------|---------|
| 所有UI按钮（16×16 / 32×32） | `SKShapeNode` + 像素化渲染 | `PixelRenderer.swift` 自研 |
| 弹弓皮筋拉伸动画 | 代码绘制贝塞尔曲线 | 动态计算拉伸状态 |
| 粒子特效（星星/冰晶/爆炸） | `SKEmitterNode` | 自定义粒子系统 |
| 裂纹纹理（裂纹冰块） | 代码生成随机裂纹线 | 每次加载随机生成 |
| Combo数字飘字 | `SKLabelNode` + 像素字体 | 配合`tff`字体文件 |
| 企鹅飞行轨迹预测线 | 虚线贝塞尔曲线 | 实时计算抛物线 |
| 背景渐变与极光动画 | Shader或层叠渐变 | 代码动画循环 |
| 心形/金币UI图标 | `SKShapeNode` | 基础几何图形 |

| 内容温和性 | 游戏内容为弹弓发射企鹅撞击冰块，无暴力血腥 |
| 操作难度 | 触控操作简单，适合低龄儿童 |
| 家长控制 | 包含内购，需要家长监管 |
| 教育价值 | 涉及物理概念（弹道、角度、力学） |

**Apple评级依据**：
- 轻微动画暴力（卡通风格企鹅撞击冰块）
- 非现实暴力场景
- 无脏话、无性内容
- 无赌博元素（内购可管控）

**家长注意事项**：
- 游戏包含应用内购买
- 建议家长开启"购买限制"功能
- 建议适当游戏时间管理

#### 回本周期分析

#### 盈亏平衡优化建议

1. **控制初期运营成本**：前3个月以自然增长为主，慎用广告投放
2. **提升ARPU**：月卡定价¥30，年卡¥228，强调性价比
3. **优化广告策略**：仅在激励广告（非插屏）场景展示广告，减少用户反感
4. **持续内容更新**：每季度更新皮肤和新关卡，维持用户兴趣
5. **用户裂变**：设置邀请奖励，利用现有用户带新用户
